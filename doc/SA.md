# 員工操作流程圖

```mermaid
flowchart TD
    A[員工登入系統] --> B{選擇操作};
    B --> C[打卡/簽到];
    B --> D[查詢個人出勤紀錄];

    C --> E{選擇打卡類型};
    E --> F[上班打卡];
    E --> G[下班打卡];
    F --> H[系統記錄時間並顯示成功訊息];
    G --> H;

    D --> I[系統查詢並顯示所有出勤紀錄];
```

# 主管審核出勤流程 (新功能)

```mermaid
flowchart TD
    subgraph "主管審核出勤流程 (新功能)"
        A[主管進入審核頁面] --> B[系統顯示待審核的異常紀錄<br>(例如：忘打卡、手動補登申請)]
        B --> C{有待審核的紀錄?}
        C -- 是 --> D[選擇一筆紀錄]
        C -- 否 --> E[顯示「無待審核項目」]
        D --> F{進行審核}
        F -- 同意 --> G[更新紀錄狀態為「已核准」]
        F -- 駁回 --> H[紀錄狀態改為「已駁回」並通知員工]
        G --> B
        H --> B
    end
```

# 結合 GPS 定位的打卡流程

```mermaid
flowchart TD
    subgraph "結合 GPS 定位的打卡流程"
        A[員工點擊打卡按鈕] --> B[向使用者請求GPS權限];
        B --> C{使用者是否授權?};
        C -- 否 --> D[顯示錯誤：請開啟定位權限];
        C -- 是 --> E[獲取目前GPS座標];
        E --> F{比對GPS座標是否在公司範圍內?};
        F -- 否 --> G[顯示錯誤：您不在公司範圍內];
        
        F -- 是 --> H{判斷當日是否已打上班卡};
        H -- 否 --> I[執行上班打卡];
        H -- 是 --> J{判斷當日是否已打下班卡};
        
        I --> K[記錄上班時間與GPS座標];
        
        J -- 否 --> L[執行下班打卡];
        J -- 是 --> M[顯示今日已完成所有打卡];
        
        L --> N[記錄下班時間與GPS座標];
        
        K --> O[顯示打卡成功];
        N --> O;
    end
```

# 員工請假流程 (新功能)

```mermaid
flowchart TD
    subgraph "員工請假流程 (新功能)"
        A[員工填寫假單] --> B[提交請假申請];
        B --> C[系統將申請發送給主管];
        C --> D{主管審核};
        D -- 同意 --> E[更新出勤紀錄為「請假」];
        D -- 駁回 --> F[申請狀態改為「已駁回」];
        E --> G[通知員工「申請已核准」];
        F --> G;
    end
```

# 主管請假管理流程 (新功能)

```mermaid
flowchart TD
    subgraph "主管請假管理流程 (新功能)"
        A[主管登入系統] --> B{選擇請假管理功能};
        B --> C[審核待處理申請];
        B --> D[查詢團隊請假紀錄];

        C --> E[系統顯示待審核列表];
        E --> F{有待審核的申請?};
        F -- 是 --> G[選擇一筆申請];
        F -- 否 --> H[顯示「無待處理申請」];
        G --> I{進行審核};
        I -- 同意 --> J[更新申請狀態為「已核准」];
        I -- 駁回 --> K[更新申請狀態為「已駁回」];
        J --> L[通知員工結果];
        K --> L;
        L --> E;

        D --> M[選擇團隊成員與時間範圍];
        M --> N[系統查詢並顯示請假紀錄];
    end
```
